# 整体代码健壮性加固 Spec

## Why
当前项目已覆盖课表、导入、设置、更新、通知/后台任务等关键能力，但仍存在少量边界条件与异常处理不一致的问题，可能导致偶发崩溃、页面错位或后台任务失效。需要一次面向全项目的“检查 + 加固”，在不引入广告/云账号强绑定、不破坏离线优先的前提下提升长期稳定性与可维护性。

## What Changes
- 建立一套可执行的“健壮性检查基线”，覆盖 UI/数据/后台任务/导入/通知等关键路径
- 收敛并统一异常处理与用户反馈策略（避免静默失败与无意义堆栈输出）
- 强化 UI 布局与数据边界保护（避免越界、重叠、空数据导致的异常渲染）
- 强化后台任务/通知相关的权限与失败兜底（权限缺失时不崩溃、可恢复、可重试）
- 清理遗留的明显风险点（如遗留 TODO、潜在空指针/越界点、非稳定排序导致的错位观感）

## Impact
- Affected specs: 课表网格布局、周次选择交互、导入解析、设置项读写、更新检查、提醒/常驻通知、后台任务调度
- Affected code:
  - feature:timetable（课表 UI、通知/后台任务、常驻通知）
  - feature:import（WebView/脚本解析、数据落库入口）
  - feature:settings（设置读写与校验、展示层）
  - feature:update（网络请求、校验、异常处理）
  - core:data / core:domain / core:ui（仓库边界、用例输入校验、公共 UI 容错）

## ADDED Requirements
### Requirement: 关键路径不得因异常输入崩溃
系统 SHALL 在以下关键路径中对异常输入与数据边界进行防御性处理，确保不发生崩溃：
- 课表网格渲染（节次、周次、星期范围、课程时长）
- 周次切换与分页展示（当前周次/总周数不一致、假期模式）
- 导入解析（解析为空、格式错误、脚本执行异常、网络异常）
- 通知与后台任务（权限缺失、调度失败、服务被系统回收）
- 设置读取（非法颜色字符串、缺失字段、旧版本数据迁移）

#### Scenario: UI 边界保护（课表网格）
- **WHEN** 课程数据出现非法节次（<=0）、非法时长（<=0）或超出最大节次
- **THEN** UI 使用安全裁剪后的值进行布局，且不发生崩溃/无限布局

#### Scenario: 假期模式标题与周次选择保持可用
- **WHEN** 当前日期对应的真实周次大于总周数
- **THEN** 顶部标题显示“假期中”，且周次下拉菜单仍可打开并选择 1..总周数

#### Scenario: 导入解析失败的可恢复性
- **WHEN** 导入解析返回空结果或抛出异常
- **THEN** 系统给出明确的失败原因提示（不含敏感信息），并允许用户重试

### Requirement: 异常处理策略一致且可测试
系统 SHALL 将关键逻辑的失败路径收敛为可测试的返回结果（例如 Result/密封类状态），避免在业务逻辑中散落 `printStackTrace()` 作为唯一处理方式。

#### Scenario: 后台任务调度异常
- **WHEN** WorkManager/AlarmManager 调度过程发生异常或权限不足
- **THEN** 系统不崩溃，并记录可诊断信息（不含隐私/凭证），并在下次合适时机自动恢复调度

## MODIFIED Requirements
### Requirement: 课表显示在冲突/异常数据下保持可读
课表网格的课程卡片在同一天节次区间重叠时 SHALL 采用可读的冲突显示策略（例如横向分栏或其他一致策略），避免卡片覆盖导致信息不可见。

### Requirement: 周次选择菜单易读且可操作
周次选择下拉菜单 SHALL 提供清晰的当前选择标识、可滚动列表与合理的宽高限制，确保在大周数下仍可快速选择目标周次。

## REMOVED Requirements
